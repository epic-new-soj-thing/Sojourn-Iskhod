# PR #278: Major LSS Overhaul

- **Author:** AMcilraith
- **Link:** https://github.com/epic-new-soj-thing/Sojourn-Iskhod/pull/278

## About The Pull Request
<details>
<summary>About The Pull Request</summary>
<hr>

This PR delivers a focused overhaul of the trade/LSS systems and quality-of-life improvements for trade interactions:

- Trade subsystem rework for reliability and clearer processing, reducing stalls and edge-case failures.
- Trade console UI/UX improvements for a smoother order flow, clearer statuses, and better error messaging.
- Numerous trade station preset cleanups (stock, categories, availability) to make offerings more consistent and sensible.
- Export price logic adjusted to better reflect item value and reduce exploits/oddities.
- Several bug fixes addressing a potential infinite loop, indentation/runtime issues, and cases where orders failed silently.
- Supporting updates to the modular computer trade program and the core trade station datum to align with the new backend behavior.
- Non-gameplay CI additions: auto-changelog generation, documentation generation, conflict checks, and screenshot test surfacing to keep contributions healthy and visible.

What players should notice:
- Trade terminals are more predictable and informative.
- Offers/stocks across stations feel cleaner and more consistent.
- Prices make more sense relative to item value.
- Fewer stalled/failed orders; errors are surfaced clearly when they happen.

What admins/maintainers should notice:
- The trade subsystem code is simpler to follow and less failure-prone.
